Version 1.4.0 (released 2014-01-01)
- New layout for the application: added the drawer, added animations, updated all interface of the application
- New design in interface for input or the edit transaction
- New design for the transaction list (colored status: gray = not reconciled, green = reconciled, red = void; orange = follow-up; duplicated = blue)
- Added function to import all currencies, in Currencies activity
- For the new database, are automatically populated all currencies and the default categories
- Transform the Categories window in a window of type ExpandibleListView
- Added new setting "Default Status": you can choose a default state to set when you do new transaction
- Added new setting "Default Payee": you can choose beetwen the choices "None"(the beneficiary must be chosen by the user) and "Last"(the application propose last payee used)
- Added check if you are opening the database may be in writing. The application does not crash more if you try to open a database in read-only
- Fixed other minor bug
- Please note this version has disabled the dashboard, as in reconstruction.
